# StarPivot 商城开发事项

> 微服务模块：`starpivot-mall-parent`（member / product / ware / order / promotion）  
> 菜单与 API 对照见 [mall.md](./mall.md)

## 1. 工程概览

### 1.1 技术栈

- Spring Boot 3 + Spring Cloud Alibaba（Nacos 注册/配置）
- MyBatis-Plus + Druid
- Spring Security + JWT（`authority-strategy: menu-permission`）
- Redis 缓存、Knife4j 接口文档

### 1.2 包结构

```
cn.org.starpivot.mall
├── config/          # 安全、MyBatis 配置
├── security/        # 商城权限菜单查询
├── portal/          # C 端商城 API（/portal 前缀，会员 JWT + Redis 购物车）
├── pms/             # 商品（controller / service / mapper / entity / domain）
├── wms/             # 仓储（仓库/库存/地区/采购/工作单）
├── oms/             # 订单（后台闭环 + C 端 portal 下单）
├── sms/             # 营销（优惠券/专题/秒杀/满减/会员价/轮播）
└── ums/             # 会员（列表/等级/积分成长/统计）
```

### 1.3 双库划分

| 库 | 用途 |
|---|---|
| `star_pivot` | 用户、角色、菜单、权限（`sys_menu`） |
| `star_pivot_product` | 商品 PMS（`pms_*`） |
| `star_pivot_ware` | 仓储 WMS（`wms_*`、`address`） |
| `star_pivot_order` | 订单 OMS（`oms_*`、`mq_message`） |
| `star_pivot_member` | 会员 UMS（`ums_*`） |
| `star_pivot_promotion` | 营销 SMS（`sms_*`，含 `sms_spu_bounds` 积分配置） |

初始化脚本：`sql/00_create_mall_database.sql`（建库）+ `sql/import_mall_databases.ps1`（导入五域 SQL）。  
省市区完整数据：`sql/address.sql`（导入 `star_pivot_ware`，`star_pivot_ware.sql` 已内置同批数据）。  
已有订单库补丁：`sql/patch_oms_refund_alert_ack.sql`（退款失败告警 `alert_ack` 字段）。

## 3. 各菜单对应数据库表设计

表名以 `sql/star_pivot_*.sql` 实际 DDL 为准（schema 源自谷粒商城，部分命名与早期设计文档不同，见 §6）。

### 3.1 商品系统 PMS

#### 分类维护（menu 25）
- `pms_category` — 三级分类：名称、父 ID、排序、图标、是否显示

#### 品牌管理（menu 26）
- `pms_brand` — 品牌名称、logo、描述、排序、显示状态
- `pms_category_brand_relation` — 分类与品牌多对多

#### 平台属性（menu 27~31）
- `pms_attr_group` — 属性分组（所属 catelog_id）
- `pms_attr_attrgroup_relation` — 属性与分组关联
- `pms_attr` — 属性总表；`attr_type`：1=规格参数（基本属性），0=销售属性

#### 商品维护 / SPU / SKU（menu 28, 130）
- `pms_spu_info` — SPU 主表：名称、品牌、分类、价格、上架状态、标题、重量等
- `pms_spu_info_desc` — SPU 详情富文本（decript 字段）
- `pms_spu_images` — SPU 图片（封面/详情图，img_type 区分）
- `pms_product_attr_value` — SPU 规格参数值（平台属性值）
- `pms_sku_info` — SKU：售价、库存、标题、条码、重量
- `pms_sku_images` — SKU 专属图片
- `pms_sku_sale_attr_value` — SKU 销售属性组合（颜色/尺寸等）

> SPU 新增/修改已在 `PmsSpuInfoServiceImpl` 中单事务级联写入以上 7 张表。

#### 评论管理（menu 262，归属内容运营）
- `pms_spu_comment` — 商品评论：SPU/SKU、用户、评分、内容、图片
- `pms_comment_replay` — 评论回复

### 3.2 优惠营销 SMS

#### 优惠券（menu 221~222）
- `sms_coupon` — 优惠券主表：类型、门槛、折扣、有效期、发行量
- `sms_coupon_spu_relation` — 优惠券关联 SPU
- `sms_coupon_spu_category_relation` — 优惠券关联分类
- `sms_coupon_history` — 用户领取/使用记录

#### 专题活动（menu 223）
- `sms_home_subject` — 专题标题、封面、起止时间
- `sms_home_subject_spu` — 专题关联商品

#### 秒杀（menu 224~225）
- `sms_seckill_promotion` — 秒杀活动
- `sms_seckill_session` — 秒杀场次（每日场次）
- `sms_seckill_sku_relation` — 秒杀 SKU：秒杀价、限购、库存
- `sms_seckill_sku_notice` — 秒杀提醒订阅

#### 积分 / 满减 / 会员价（menu 226~228）
- `sms_spu_bounds` — SPU 积分赠送配置（买赠积分、成长值）
- `sms_sku_full_reduction` — SKU 满减（满 X 减 Y）
- `sms_sku_ladder` — SKU 阶梯折扣（满件打折）
- `sms_member_price` — SKU 会员阶梯价

#### 首页轮播（menu 261）
- `sms_home_adv` — 首页广告：图片、链接、排序、上下架

### 3.3 库存系统 WMS

#### 仓库 / 库存（menu 132, 144）
- `wms_ware_info` — 仓库名称、地址、负责人、电话
- `wms_ware_sku` — 仓库 SKU 库存、锁定库存、销量

#### 地区管理（menu 168）
- `address` — 省市区乡镇四级联动（独立表，非 wms 前缀；数据见 `sql/address.sql`）

#### 库存工作单（menu 273）
- `wms_ware_order_task` — 出库/入库工作单
- `wms_ware_order_task_detail` — 工作单 SKU 明细

#### 采购（menu 271~272）
- `wms_purchase` — 采购单主表：单号、供应商、审核状态
- `wms_purchase_detail` — 采购明细 SKU、数量、单价

> 注：早期设计中的 `wms_purchase_demand`（采购需求）在现有 schema 中未建表，采购需求功能待产品确认后补表或复用采购单流程。

### 3.4 订单系统 OMS

#### 订单查询（menu 201）
- `oms_order` — 订单主表：订单号、会员、地址、金额、状态
- `oms_order_item` — 订单明细 SKU
- `oms_order_operate_history` — 操作日志（发货、取消、确认收货）

#### 退货处理（menu 202）
- `oms_order_return_apply` — 退货申请（含 SKU 明细字段，无独立 return_item 表）
- `oms_order_return_reason` — 退货原因字典

#### 订单设置（menu 203）
- `oms_order_setting` — 订单自动确认、自动完成等规则

#### 支付 / 退款（menu 204~205）
- `oms_payment_info` — 支付流水（支付宝/微信/Mock）
- `oms_refund_info` — 退款流水（含 `alert_ack` 失败告警已读）
- **已对接**：支付宝/微信原路退款、微信退款异步通知、渠道查询对账、失败重试、定时同步退款中状态、B 端失败告警条

#### 基础设施
- `mq_message` — 本地消息表（Outbox 已实现 `OmsLocalMessageOutboxService`，见 [mall-p3-distributed.md](./mall-p3-distributed.md)）
- `undo_log` — Seata 回滚日志（表占位，Seata 暂缓）

### 3.5 用户系统 UMS

#### 会员列表（menu 241）
- `ums_member` — 手机号、昵称、头像、等级、积分、成长值
- `ums_member_receive_address` — 会员收货地址
- `ums_member_login_log` — 登录日志
- `ums_member_collect_spu` — 收藏商品
- `ums_member_collect_subject` — 收藏专题

#### 会员等级（menu 242）
- `ums_member_level` — 等级名称、成长值门槛、折扣、免邮、积分倍率

#### 积分成长（menu 243）
- `ums_integration_change_history` — 积分变动记录
- `ums_growth_change_history` — 成长值变动记录

#### 统计信息（menu 244）
- `ums_member_statistics_info` — 会员统计快照（也可 SQL 聚合）

## 7. 开发规范

1. **Controller 路径**：B 端统一 `/mall/{resource}`；C 端统一 `/portal/{resource}`，完整 URL 分别为 `/api/v1/mall/...`、`/api/v1/portal/...`
2. **权限注解**：与 `sys_menu.perms` 一致，如 `@PreAuthorize("hasAuthority('mall:category:add')")`
3. **分页**：入参继承 `PageRequest`，返回 `PageResponse<T>`
4. **异常**：业务异常抛 `BizException`，错误码见 `starpivot-common`
5. **新增表**：先改对应域 `sql/star_pivot_{product,ware,order,member,promotion}.sql`，再 `node scripts/gen_entities.js` 生成 Entity，手写 Mapper XML（复杂查询）或继承 `BaseMapper`
6. **菜单**：页面菜单以 `sql/sys_menu.sql` / `sql/star_pivot.sql` 为准；已有库若 ID 冲突执行 `migrate_mall_menus_v2.sql`
7. **C 端鉴权**：公开接口免登录；会员接口 JWT 角色 `MEMBER`，不走 `sys_menu`
